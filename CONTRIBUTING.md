# Contributing to AI Prompt Security Trainer

Thanks for wanting to help make this repo better! This project exists to teach AI security in plain, beginner-friendly English — so contributions should keep that spirit.

## How to Contribute

1. **Fork** this repository
2. **Create a branch** for your change: `git checkout -b add-topic-name`
3. **Make your change**
4. **Commit** with a clear message: `git commit -m "Add example for XSS in AI-generated code"`
5. **Push** and open a **Pull Request**

## What We're Looking For

- ✅ New real-world examples (anonymized/generalized — no real company names or real leaked secrets)
- ✅ Corrections to inaccurate or outdated information
- ✅ Clearer explanations of existing topics
- ✅ New diagrams (Mermaid preferred — renders natively on GitHub)
- ✅ Translations
- ✅ New defense tools or techniques worth mentioning

## Writing Style Guide

To keep the repo consistent and beginner-friendly:

- **Keep it simple.** Avoid unnecessary jargon. If you must use a technical term, explain it in plain English the first time it appears.
- **Follow the existing structure** for each topic:
  1. What is the problem?
  2. What are its types/common variations?
  3. A simple, practical example
- **Use realistic but fake examples.** Never include real secrets, real vulnerable production code, or real company/personal data.
- **Diagrams:** use [Mermaid](https://mermaid.js.org/) syntax in fenced code blocks — it renders automatically on GitHub, no image files needed.
- **Code examples:** keep them short (under ~15 lines) and clearly mark vulnerable vs. fixed versions.

## What We Won't Accept

- ❌ Working exploit code intended for attacking real, non-consenting systems
- ❌ Real leaked secrets, credentials, or private data — even as "examples"
- ❌ Content promoting attacks without an accompanying defense/fix
- ❌ Unattributed copy-pasted content from other sources

## Reporting Issues

Found something inaccurate, outdated, or confusing? Please [open an issue](../../issues) — include:
- Which section/topic it's about
- What's wrong or unclear
- A suggested fix, if you have one

## Code of Conduct

Be respectful. This is a learning resource — assume good faith, especially with beginners asking questions.
